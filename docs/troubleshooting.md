# 故障排查

## 连接不上

先确认 URL 是否完全一致：

```text
http://mcp.wjr.world/mcp-servers/wjr-quant-sse/sse
```

常见错误：

- 把 `http` 写成 `https`
- 少写 `/sse`
- 多写空格
- 客户端不支持 SSE MCP
- 本地网络或公司网络拦截了外部连接

## 认证失败

确认请求头是：

```text
Authorization: Bearer <你的AI网关消费者Token>
```

注意：

- `Authorization` 首字母可以大小写不敏感，但建议照抄。
- `Bearer` 后面要有一个空格。
- 不要把尖括号一起复制进真实 Token。
- Token 过期或停用后，需要重新生成。

## 工具列表为空

可以尝试：

1. 重启客户端。
2. 删除旧 MCP 配置后重新添加。
3. 确认客户端选择的是 SSE 类型。
4. 确认 Token 没有填错。
5. 等待 30 秒后重新刷新工具列表。

## 工具能看到但调用失败

常见原因：

- `strategy_id` 为空或不存在。
- `limit` 不是数字。
- `limit` 太大。
- 客户端自动生成的参数格式不符合工具要求。

建议先问：

```text
查看有哪些可用策略。
```

拿到策略 ID 后，再问：

```text
查看这个策略的最新信号，限制返回 10 条。
```

## 返回内容和预期不一致

本服务只返回已经发布的策略资产。如果你期待的是实时荐股、训练模型、生成新信号或自动交易，这些都不属于本服务范围。

## 仍然无法使用

请记录以下信息后再排查：

- 使用的客户端名称和版本
- MCP 配置片段，但不要包含真实 Token
- 报错文字
- 发生时间
- 是否能看到工具列表
- 哪个工具调用失败
